⚙️ The Gradient Function Machine

Understanding a function that outputs slope instead of height

A New Kind of Function

Most functions you've seen take an x-value and give back a y-value:

f(x) = 2x + 3 → input x, output a height

But our differential equation is different. It's a rule that takes a point and gives back a slope:

Δy/Δx = y → input a point, output a slope

This might feel weird at first — and that's okay! Let's build intuition with a machine metaphor.

Meet the Gradient Function Machine

Think of the slope rule as a machine. You feed it a point, and it tells you the slope — the gradient — at that location.

Our particular gradient function machine has a very simple rule: the slope equals the y-value. That's it!

What does this mean in plain English?

When the curve is high up (big y), it's steep (big slope). When the curve is low (small y), it's gentle (small slope). The higher you are, the faster you climb.

The Pattern

Different rules produce different slopes! The gradient function machine can depend on y alone (slope = y), x alone (slope = 3x), or both x and y (slope = x - 2y + 1).

With slope = y, only the height matters — the x-value is irrelevant. With slope = 3x, it's the opposite — vertical position doesn't matter, only where along the x-axis you are (scaled by 3). And slope = x - 2y + 1 mixes both, with y having stronger influence.

🏔️ Think of it this way: A differential equation is just a rule that assigns a slope to every point. What determines the slope depends entirely on the rule. For our Path Builder Challenge we'll use Δy/Δx = y — where the higher you are, the steeper you climb — but now you know that's just one possible gradient function.

Why This Matters

In the Path Builder Challenge, you'll use this slope machine at every step:

The Euler Method Loop

1. Look at your current point → read its y-value

2. Feed y into the slope machine → get the slope

3. Use the slope to take a step → arrive at a new point

4. Repeat from step 1

The gradient function machine is the engine that drives every step. Now that you understand how it works, you're ready to build paths!
